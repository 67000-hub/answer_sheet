# 答题卡识别系统

基于 Python + OpenCV 的答题卡自动识别与判分系统，支持**圆形气泡、矩形括号两种选项框样式**，支持**单选、4选项、多列布局**的答题卡，并支持**主观题 OCR 文字识别**和**模板批量处理**。

## 功能特性

- 自动检测答题卡外框并透视变换矫正倾斜
- 自适应识别**圆形气泡选项**（支持带字母的空心气泡）
- 自适应识别**矩形括号选项**（如 `[A][B][C][D]` 格式）
- **智能题号过滤**：自动排除左侧题号干扰（基于行规律性检测）
- 自动识别多列布局（1 列、2 列、3 列等）
- 相对阈值判断填涂状态，不受光照影响
- 未填涂题目自动识别（不画圈、不计分）
- 图形化展示识别结果，终端打印答案与分数
- **结果图片自动缩放**：大图自动缩放到适合屏幕的尺寸（最大 1000×700）
- **主观题 OCR 识别**：手动框选主观题区域，OCR 自动转文字
- **主观题多区域框选**：每道主观题可框选多个区域
- **答题卡模板**：框选一次，批量处理多份相同排版的答题卡

## 环境依赖

- Python 3.7+
- OpenCV (`opencv-python`)
- NumPy
- Pillow
- tkinter（Python 内置）
- **可选**：RapidOCR / PaddleOCR（主观题 OCR 识别需要）

### 安装依赖

```bash
pip install -r requirements.txt
```

### 安装 OCR 依赖（主观题识别用）

```bash
pip install rapidocr-onnxruntime
```

> 推荐使用 **RapidOCR**（ONNX Runtime 版本），兼容性好，安装简单。
> 也支持 PaddleOCR：`pip install paddleocr paddlepaddle`（Python 3.12 及以下）。
> 第一次运行时会自动下载模型文件。

## 使用方法

### 运行程序

```bash
python get_answer.py
```

### 选择答题卡类型

启动后会弹出对话框选择答题卡类型：

| 选项 | 说明 |
|------|------|
| **Yes** | 矩形括号选项框（`[A][B][C][D]` 格式） |
| **No** | 圆形气泡选项框（●/○ 格式） |

### 三种运行模式

随后弹出 **"Select Run Mode"** 对话框：

| 按钮 | 模式 | 说明 |
|------|------|------|
| **是 (Yes)** | 使用模板 | 选择已有模板 + 选择多张图片，自动批量处理 |
| **否 (No)** | 手动模式 | 每次手动框选区域（原有功能） |
| **取消 (Cancel)** | 创建模板 | 在模板图片上框选区域，保存为模板供后续使用 |

### 手动模式操作步骤

1. **选择图片**：弹出文件选择对话框
2. **选择答题卡类型**：圆形气泡 / 矩形括号
3. **输入答案**：填写每题正确答案（逗号或空格分隔），如 `B,D,A,C`
4. **选择是否包含主观题**：
   - **否**：自动检测全图答题卡
   - **是**：
     1. 输入主观题数量
     2. 依次框选每道主观题区域
     3. 框选客观题区域
5. **查看结果**：终端输出答案和得分，窗口显示标注图（自动缩放到适合屏幕的尺寸）

### 创建模板操作步骤

1. 选择一张模板答题卡图片
2. 选择答题卡类型
3. 输入正确答案
4. 选择是否包含主观题
5. 输入模板名称
6. 依次框选主观题区域 + 客观题区域
7. 模板自动保存到 `templates/` 目录
8. 同时处理模板图片本身

### 使用模板操作步骤

1. 从列表中选择一个模板
2. 选择一张或多张答题卡图片（支持多选）
3. 程序自动用模板中的 ROI 坐标框选各区域
4. 逐张输出结果

### 框选操作说明

| 按键 | 功能 |
|------|------|
| 鼠标拖拽 | 画出选框 |
| 回车 (Enter) | 确认当前区域（多区域模式下添加区域） |
| N 键 | 结束本道题的框选（多区域模式） |
| R 键 | 删除最后一个区域 / 重新选择 |
| C 键 | 清除当前正在画的矩形 |
| ESC 键 | 取消 |

> 所有框选窗口均自动缩放到最大 1000×700 像素，防止超出屏幕。选择的坐标会自动映射回原图尺寸。

### 识别结果说明

- **蓝色框/圈**：学生填涂的选项
- **绿色框/圈**：答对的正确答案
- **红色框/圈**：答错的正确答案
- 未填涂的题目不画蓝色框/圈

## 支持的答题卡格式

| 项目 | 说明 |
|------|------|
| 题型 | 单项选择题 |
| 选项数 | 4 个（A/B/C/D） |
| 布局 | 单列、多列均可（自动检测列数） |
| 选项框样式 | 圆形气泡 / 矩形括号（`[A][B][C][D]`） |
| 填涂方式 | 实心填涂 / 空心带字母均可 |
| 题号干扰 | 自动过滤左侧题号（11、12 等两位数也能识别） |
| 批量处理 | 支持模板批量处理多份相同排版的答题卡 |

## 识别原理

### 整体流程

```
答题卡照片
    ↓
灰度化 + 高斯模糊 + Canny边缘检测
    ↓
找最大四边形轮廓 → 透视变换矫正
    ↓
Otsu 自动阈值二值化（反色：填涂=白）
    ↓
选项框检测（圆形气泡 / 矩形括号）
    ↓
行规律性过滤 → 排除题号等干扰
    ↓
按 x 坐标分列（自动检测列数）
    ↓
每列内 top-to-bottom 排序 → 每 4 个一组 = 一题
    ↓
每组内 left-to-right 排序 → A/B/C/D
    ↓
测填涂量 → 相对比较判定填涂
    ↓
对比正确答案 → 评分 + 可视化输出（自动缩放显示）
```

### 关键算法

1. **透视变换**：用 `approxPolyDP` 近似四边形，`getPerspectiveTransform` 做单应性变换，将倾斜拍摄的答题卡拉正
2. **圆形气泡检测**：形态学闭运算补全断裂 + RETR_TREE 层级检测 + 动态尺寸筛选（基于宽度中位数）+ 中心去重
3. **矩形括号检测**：轮廓特征分类（括号形 vs 实心矩形）+ 左右括号配对 → 合并为完整矩形区域
4. **题号智能过滤**：基于行规律性检测，主动寻找每行中大小一致、间距均匀的 4 个一组选项框，自动排除左侧题号干扰
5. **多列布局**：根据气泡 x 坐标间距自动分列，列内排序分组后按列拼接
6. **填涂判定**：同一题内比较各选项的填涂像素量，最高值明显高于其他平均值（1.3 倍 + 50 像素）则判为填涂
7. **自动缩放显示**：所有显示图片自动缩放到最大 1000×700 像素，框选坐标自动映射回原图

## 常见问题

### Q: 检测到的题目数不对？
A: 检查 `columns_debug` 窗口，看分列是否正确。如果列间距阈值不合适，可以调整 `config.py` 中的 `COL_GAP_MULTIPLIER` 系数。

### Q: 填涂识别不准？
A: 看终端的 `[DEBUG]` 输出，检查每题的 totals 值。如果最大值和其他值差距不大，可以调小判定系数（`FILL_RATIO_THRESHOLD` 改为更小的值）。

### Q: 气泡/选项框漏检？
A: 检查对应的调试窗口。圆形气泡可调整形态学核大小或尺寸筛选范围；矩形括号可调整括号特征阈值（`RECT_BRACKET_*` 配置项）。

### Q: 矩形括号选项识别不准？
A: 矩形括号通过左右括号配对识别，确保括号清晰完整。如果括号印刷不完整，可适当放宽 `RECT_BRACKET_FILL_MIN/MAX` 范围。

### Q: 题号被误检测为选项框？
A: 已内置行规律性过滤算法，自动排除左侧题号干扰。如果仍有误检，可以：
- 检查 `pattern filter` 的 debug 输出，看是否正确识别了选项框组
- 确认题号是否完全位于选项框组的左侧且间距差异明显

### Q: 图片显示超出屏幕？
A: 已自动缩放到最大 1000×700 像素。如需调整，修改 `config.py` 中的 `DISPLAY_MAX_WIDTH` 和 `DISPLAY_MAX_HEIGHT`。

### Q: 主观题 OCR 识别不准？
A: 确保框选区域只包含该题文字，不要包含太多空白区域。手写体识别效果可能不佳，建议用于印刷体题目。

### Q: 提示未安装 OCR 引擎？
A: 执行 `pip install rapidocr-onnxruntime` 安装（推荐，兼容性好）。
也可以安装 PaddleOCR：`pip install paddleocr paddlepaddle`。
如果不需要主观题识别功能，选择"仅客观题"模式即可。

### Q: 模板处理时区域偏移？
A: 模板基于像素坐标，要求所有答题卡图片的**分辨率和排版完全一致**。如果拍摄角度或缩放不同，区域会对不准。建议用扫描件或固定位置拍摄。

### Q: 窗口标题乱码？
A: 窗口标题已改为英文以避免编码问题，不影响功能使用。
